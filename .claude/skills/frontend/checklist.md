# Frontend Checklist

Use this checklist before considering frontend work complete.

## Scope
- Only the intended slice was implemented
- No accidental scope expansion was introduced
- The UI matches the defined feature or slice intent

## Structure
- Existing layout and component patterns were reused where possible
- New components were introduced only when justified
- File structure remains understandable

## States
- Empty states handled where relevant
- Loading states handled where relevant
- Error states handled where relevant
- Success or completion feedback handled where relevant

## Clarity
- Navigation is clear
- Labels and actions are understandable
- The interface is not visually overloaded
- The result improves usability instead of adding decoration

## Consistency
- UI is consistent with the rest of the application
- Spacing, layout logic, and interaction patterns are not arbitrary
- The implementation respects project UI rules

## Technical quality
- No obvious dead UI code was introduced
- No unnecessary dependencies were added
- The implementation is reasonably maintainable
- The result is ready for QA
