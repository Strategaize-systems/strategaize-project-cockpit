# SLC-004 Features Table

## Slice ID
SLC-004

## Name
Features Table

## Related Feature
FEAT-002 Features View

## Purpose
Render the project feature index in a structured and readable way.

## Goal
Provide a clean feature-level visibility layer.

## Included in scope
- feature table or list
- feature ID
- feature name
- feature status
- priority
- notes

## Explicitly out of scope
- feature editing workflows
- drag-and-drop boards
- timeline views

## Expected result
Features are easier to review than by reading raw markdown alone.

## Acceptance criteria
- features render clearly
- status is easy to interpret
- feature review becomes faster

## Risks
- overcomplicating the first features view
- unclear source formatting expectations

## Priority
high

## Status
planned
