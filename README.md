# Strategaize Project Cockpit

Strategaize Project Cockpit is the first real project built under Strategaize Dev System.

It is an internal operational cockpit designed to make project visibility faster, clearer, and less dependent on manually opening raw markdown files.

## Purpose

The cockpit exists to provide a structured internal view of project information such as:

- project state
- feature status
- slice status
- known issues
- releases
- migrations
- decisions
- skill improvement findings

## Why this project matters

This is not only a useful internal tool.

It is also the first practical validation project for Strategaize Dev System itself.  
That means the project should prove that the broader development system is usable in real work.

## V1 focus

V1 is intentionally limited.

It focuses on:
- read-oriented visibility
- operational clarity
- fast resume capability
- small internal usability improvements

V1 is not intended to become a full project management platform.

## Delivery mode

Internal Tool

## Preferred stack direction

- Next.js
- Tailwind CSS
- shadcn/ui
- Docker
- Hetzner
- Coolify

## Core V1 areas

- Project Overview
- Features View
- Slices View
- Bugs / Issues View
- Releases / Migrations View
- Decisions / Skill Improvements View

## Project role inside the larger system

Strategaize Project Cockpit should later help support multiple future projects by making their records easier to review through a clear internal UI.

## Important constraint

V1 must stay small, practical, and fast to deliver.  
The main goal is to make project status operationally visible first.
