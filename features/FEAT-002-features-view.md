# FEAT-002 Features View

## Feature ID
FEAT-002

## Name
Features View

## Purpose
Provide structured visibility into the feature-level scope of the selected project.

## Problem
Feature scope is often unclear once multiple additions, changes, and ideas accumulate over time.

## Goal
Make feature status transparent and easy to review.

## Included in scope
- feature ID
- feature name
- feature status
- priority
- short notes
- clear list or table presentation

## Explicitly out of scope
- drag-and-drop boards
- advanced PM workflows
- feature editing interface in V1
- timeline forecasting

## Data sources
- `features/INDEX.md`
- later individual `FEAT-...` records

## Expected UI outcome
A structured list or table that makes it easy to see:
- what is planned
- what is active
- what is done
- what is blocked
- what is deferred

## Acceptance criteria
- features are clearly listed
- status is easy to understand visually
- priorities are visible
- the feature list is more usable than manually opening the raw markdown file

## Priority
high

## Status
planned
