# FEAT-006 Decisions / Skill Improvements View

## Feature ID
FEAT-006

## Name
Decisions / Skill Improvements View

## Purpose
Provide visibility into important project decisions and recurring lessons that should improve the broader development system.

## Problem
Important decisions and repeated delivery learnings are easy to lose when they are buried in documentation or chat history.

## Goal
Make strategic technical and process learnings clearly reviewable.

## Included in scope
- decision records
- skill improvement records
- identifiers where present
- status where relevant
- short reason or observation visibility
- simple structured view

## Explicitly out of scope
- automated system evolution
- direct editing of all decision records in V1
- governance workflow
- approval engine

## Data sources
- `docs/DECISIONS.md`
- `docs/SKILL_IMPROVEMENTS.md`

## Expected UI outcome
A view that makes it easy to answer:
- what decisions shaped this project?
- what repeated problems should improve the dev system?
- what has already been recognized as a pattern?

## Acceptance criteria
- decisions are visible
- improvement records are visible
- repeated learnings are easier to review
- the view supports long-term improvement and handoff

## Priority
medium

## Status
planned
